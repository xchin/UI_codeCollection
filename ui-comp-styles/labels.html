Floating Labels

Use with caution; please test well before pushing to production.

Instructions for use:

To add the floating label style to inputs, the addition of the floatingLabel class is required in addition to a data-label-content attribute that should contain the same text as your actual <label> element as well as the existing basic form input classes from above. To add the functionality, of the floating label, the addition of the inputs-selects.js or the compiled lib.js is needed.

The JS, when executed on the client, loops through all the instances of the floatingLabel class and adjusts the input's according to an existence of a provided defaultValue or not. This means that all inputs start with the label in the "up" position, and then after the loops adjusts it according to what it finds.

If performance is critical , or you do not want any shifting of the inputs between server render and JS execution, then you are welcome to manually add the _active postfix to the class. This tells the component that you will be responsible for manually adding hasValue class when an input has a pre-existing defaultValue or not. If you take this route, you are wholly responsible for managing the server-rendered state.

Simple Floating Label Input

Code

								<div class="floatingLabel [has-error]"
								data-label-content="Text">
								<label for="form-control-error" id="simple-form-control">Text</label>
								<input class="form-control form-control"
								type="text" id="simple-form-control" />
								</div>

Complex Floating Label Input

Please note that these "complex", floating-label components syntactically differ from that of the non-floating-label components. Also note that the error icon is required, but will only show when has-error is added to container.
